Sistema Web – APAE de Tijucas
1. Apresentação do Projeto

Este projeto foi desenvolvido no contexto do Projeto Integrador Extensionista, da disciplina de Responsive Web Developer.

A proposta consiste no desenvolvimento de um sistema web institucional voltado à divulgação e apoio às atividades da APAE de Tijucas, com foco na promoção da causa, organização de informações e incentivo à participação da comunidade.

O sistema foi desenvolvido com o objetivo de oferecer uma solução simples, acessível, responsiva e de baixo custo de manutenção.

2. Estrutura do Projeto (Parte 1 e Parte 2)

O projeto foi desenvolvido em duas etapas:

🔹 Parte 1 – Protótipo

Na primeira etapa foi desenvolvido o protótipo do sistema contendo:

- Estrutura das páginas em HTML;

- Estilização com CSS;

- Layout responsivo;

- Simulação de formulários;

- Organização visual da navegação.

Nesta fase, o sistema era estático e não possuía integração com banco de dados.

🔹 Parte 2 – Sistema Implementado

Na segunda etapa o sistema foi evoluído para uma aplicação funcional, incluindo:

- Integração com backend em Node.js;

- Criação de API REST com Express;

- Integração com banco de dados PostgreSQL;

- Cadastro de usuários;

- Sistema de login com autenticação;

- Registro real de doações no banco;

- Criptografia de senha utilizando Bcrypt.

Nesta etapa o projeto deixou de ser apenas visual e passou a ter persistência de dados e validação real.

3. Sobre a Instituição

A APAE de Tijucas é uma Organização Não Governamental (ONG) sem fins lucrativos que atua no atendimento a pessoas com deficiência intelectual e múltipla.

A instituição oferece suporte educacional, terapêutico e social, promovendo inclusão, desenvolvimento e melhoria da qualidade de vida dos assistidos, além de apoiar suas famílias.

Seu trabalho é essencial para o fortalecimento da inclusão social no município e região.

4. Objetivo do Sistema

O sistema web foi desenvolvido com a finalidade de:

- Divulgar a instituição e sua atuação;

- Apresentar eventos promovidos;

- Disponibilizar canal para doações;

- Organizar informações institucionais;

- Facilitar o engajamento da comunidade;

- Permitir cadastro e autenticação de usuários;

- Registrar doações com persistência em banco de dados.
